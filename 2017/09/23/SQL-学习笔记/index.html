SQL 学习笔记

Posted on 2017-09-23 | In 数据库 |

SQL基础教程（第2版）学习笔记。

数据库和数据库管理系统

将大量数据保存起来，通过计算机加工而成的可以进行高效访问的数据集合称为数据库(Database，DB)。

用来管理数据库的计算机系统称为数据库管理系统（Database Management System，DBMS）。

DBMS 分类

  1. 层次数据库（Hierarchical Database，HDB）：把数据通过层次结构（树形结构）的方式表现出来。

  2. 关系数据库（Relational Database，RDB）：采用由行和列组成的二维表来管理数据。使用 SQL（Structured Query Language，结构化查询语言）对数据进行操作。

    这种类型的 DBMS 称为关系数据库管理系统（Relational Database Management System，RDBMS）。

  3. 对象数据库（Object Oriented Database，OODB）：把数据以及对数据的操作集合起来以对象为单位进行管理。

  4. XML 数据库（XML Database，XMLDB）：对 XML 形式的大量数据进行高速处理。

  5. 键值存储系统（Key-Value Store，KVS）：保存查询所使用的主键(Key)和值(Value)的组合的数据库。

RDBMS 代表

  1. Oracle Database
  2. SQL Server
  3. DB2
  4. PostgreSQL
  5. MySQL

关系数据库管理系统结构

RDBMS 通常使用客户端 / 服务器的系统结构。

  • 服务器：用来接收其他程序发出的请求，并对该请求进行相应处理的程序（软件），或者是安装了此类程序的设备（计算机）。
    客户端： 向服务器发出请求的程序（软件），或者是安装了该程序的设备（计算机）。

操作数据库时通过从客户端向服务器端发送SQL语句来实现数据库的读写操作。

关系数据库结构

关系数据库采用被称为数据库表的二维表来管理数据。

一个数据库中可以存储多个表。

  • 表：管理数据的二维表。

  • 字段：表的列（垂直方向），代表了保存在表中的数据项目。

  • 记录：表的行（水平方向），相当于一条数据。

    关系数据库必须以行为单位进行数据读写

  • 单元格：行和列交汇的方格，每个单元格只能输入一个数据。

SQL 语句种类

SQL 是一种专门用来与数据库通信的语言。用关键字、表名、列名等组合而成的一条语句来
描述操作的内容。

SQL 语句可以分为三类：

  1. DDL（Data Definition Language，数据定义语言）用来创建或者删除数据库以及数据库中的表等对象。
    1. CREATE：创建数据库和表等对象；
    2. DROP：删除数据库和表等对象；
    3. ALTER：修改数据库和表等对象的结构。
  2. DML（Data Manipulation Language，数据操纵语言）用来查询或者变更表中的记录。
    1. SELECT：查询；
    2. INSERT：插入；
    3. UPDATE：更新；
    4. DELETE：删除。
  3. DCL（Data Control Language，数据控制语言）用来确认或者取消对数据库中的数据进行的变更、对 RDBMS 的用户是否有权限操作数据库中的对象（数据库表等）进行设定。
    1. COMMIT：确认变更；
    2. ROLLBACK：取消变更；
    3. GRANT：赋予用户操作权限；
    4. REVOKE: 取消用户的操作权限。

最后注释有两种：-- 和 /* */。

  1. 行注释书写在 -- 之后，只能写在同一行；
  2. 多行注释书写在 /* 和 */ 之间，可以跨多行。

创建、使用数据库

  1. 新建数据库：CREATE DATABASE <数据库名称>;

  2. 使用数据库：USE <数据库名称>;

  3. 显示所有数据库：SHOW DATABASES;

    其它 SHOW 语句：

    • 显示一个数据库中所有的表：SHOW TABLES;
    • 显示一个表中的所有列信息：SHOW COLUMNS FROM <表名称>;； DESCRIBE 可作为 SHOW COLUMNS FROM 的快捷方式。
    • 显示服务器状态信息：SHOW STATUS;
    • 显示授予用户（所有用户或特定用户）的安全权限：SHOW GRANTS;
    • 显示服务器错误或警告消息：SHOW ERRORS 和 SHOW WARNINGS;

只能使用半角英文字母、数字、下划线(_)作为数据库、表和列的名称 。

创建「Product」表时用到了数据类型，常见的四种数据类型：

  1. INTEGER：指定存储整数，不能存储小数。

  2. CHAR：指定存储字符串（不可变长字符串的形式）。在括号中指定可以存储的字符串的最大长度。当列中存储的字符串长度达不到最大长度的时候，自动使用半角空格进行补足。

  3. VARCHAR：与 CHAR 不同的是以可变长字符串的形式存储字符串，即使字符数未达到最大长度，也不会用半角空格补足

    特别的：Oracle 中使用 VARCHAR2 型。

  4. DATE：指定存储日期（年月日）。

    特别的：Oracle 中使用的 DATE 型还包含时分秒。

「Product」表中还使用了约束，如 NOT NULL。约束是列中存储的数据进行限制或者追加条件。

  • NOT NULL 约束：NULL 代表空白（无记录）。加上表示否定的 NOT 后表示该列不能输入空白。
  • PRIMARY KEY 约束：唯一标识数据库表中的每条记录。表中只能有一个主键且主键列不能包含 NULL 值。

最后指定了表的编码格式为 UTF8：DEFAULT CHARSET UTF8 。

删除、更新表

  1. 删除表：DROP TABLE <表名>;

  2. 添加表列：ALTER TABLE <表名> ADD COLUMN <列的定义>;，如向「Product」表中添加一列：ALTER TABLE Product ADD COLUMN product_name_pinyin VARCHAR(100);。

  3. 删除表列：ALTER TABLE <表名> DROP COLUMN <列名>;，如删除「Product」表中一列：ALTER TABLE Product DROP COLUMN product_name_pinyin;。

    特别的，添加和删除表列时：

    1. Oracle 和 SQL Server 在添加表列时不用写 COLUMN，Oracle 在删除表列时不用写 COLUMN；
    2. 在 Oracle 中同时添加/删除多列的时候，可以使用括号：ALTER TABLE <表名> ADD/DROP (<列名>，<列名>，......);。

  4. 更改表名：标准 SQL 没有更改表名的语法，各数据库语法如下

    • Oracle、PostgresSQL：ALTER TABLE Poduct RENAME TO Product;；
    • DB2：RENAME TABLE Poduct TO Product;；
    • SQL Server：sp_rename 'Poduct', 'Product';；
    • MySQL：RENAME TABLE Poduct to Product;。

查询数据

SELECT 语句

格式：SELECT <列名>，...... FROM <表名>;。

  1. 查询所有列：SELECT * FROM <表名>;。

  2. 为列设置别名：可以使用 AS 关键字为列设置别名。

    如果要设置中文别名，需要将中文用双引号扩起来。

  3. 书写常数：SELECT 可以书写常数。

    使用字符串或者日期常数时，需要使用单引号括起来。

WHERE 子句

SELECT 语句通过WHERE 子句来指定查询数据的条件。

格式：SELECT <列名>, ...... FROM <表名> WHERE <条件表达式>;

WHERE 子句必须紧跟在 FROM 子句之后。

算术运算符

SQL 语句中可以使用计算表达式。

将 sale_price 的两倍放在了 sale_price_x2 列中。

SQL 支持加减乘除，并且所有包含 NULL 的计算，结果都为 NULL。

比较运算符

在 WHERE 子句中通过使用比较运算符可以组合出各种各样的条件表达式。= 也是一种比较运算符。主要比较运算符如下：

  1. = ：相等；
  2. <> ：不相等；
  3. >= ：大于等于；
  4. > ：大于；
  5. <= ：小于等于；
  6. < ：小于。

注意：

  1. 字符串类型的数据原则上按照字典顺序进行排序，
  2. 不能对 NULL 使用比较运算符（提供了 IS NULL 和 IS NOT NULL 运算符）。
