Django 注册和登陆功能实现

Posted on 2018-02-05 | In Django |

Django 提供了用户身份验证机制，可实现注册和登录功能。

settings.py 文件设置

找到 INSTALLED_APPS 列表，确保含有下面两项：

  1. django.contrib.auth 用于访问 Django 提供的身份验证系统，
  2. jango.contrib.contenttypes 使 auth 应用跟踪数据库中的模型。

密码

Django 的 auth 应用并不存储明文密码，而是存储密码哈希值，默认使用 PBKDF2 算法将密码加密。

用户模型

Django 默认有一套用户模型（django.contrib.auth.models.User），这里就使用此模型构建网站的用户功能。

django.contrib.auth.models.User 的默认属性有：username、first_name、last_name、email、password、is_staff、is_active、is_superuser、last_login、date_joined 等。

如果还需要对用户添加其它属性，可在应用的 models.py 文件中新定一个模型，再与 User 模型建立一对一关系（OneToOneField）。

这里不再添加其它模型。

用户表单

定义用户表单用于在 HTML 中相应的模型字段。在应用的 forms.py 文件中定义一个 UserForm 类。

UserForm 类中重新定义 password 属性，使用 PasswordInput() 组件，使得用户在输入密码时不可见。同时还定义了密码最短 8 位，最长 30 位，且为必填项。

Meta 类中还定义了一些其它属性，包括关联模型为 User，表单中显示（fields）的字段为 username、email 和 password；还可以使用 exclude 指定要在表单排除的字段。

注册功能

注册视图

在应用的 views.py 文件中定义一个 register() 视图来处理表单数据。

  1. 局部布尔变量 registered 只有在注册成功后才改为 True，在 Html 模版中会用到。
  2. 如果是 HTTP POST 请求，就处理表单数据：
    1. 通过 UserForm(data=request.POST) 来获取表单中的所有数据；
    2. 如果表单数据可用，就通过 save() 方法将数据保存到数据库中，通过 set_password 方法计算密码哈希值并保存到数据库中；
    3. 将布尔变量 registered 改为 True。
    4. 如果表单数据无效，在终端中打印错误信息。
  3. 如果不是 HTTP POST 请求，渲染空的表单数据。
  4. 最后通过 render 渲染模版，将表单数据和布尔变量以字典形式传入 render。

注册页面的 HTML 模版

  1. 通过布尔变量 registered 判断注册是否成功，以此使界面显示不同内容；
  2. Django 要求表单中必须有 CSRF（跨站请求伪造）令牌（{% csrf_token %}），用于增强提交表单后发送 HTTP POST 请求的安全性。
  3. as_p 模板函数使得段落（HTML <p> 标签）中显示各个表单元素，一行显示一个表单元素。

最后要添加 URL 映射，并且在 HTML 模版中添加注册链接。

登录功能

登录视图

在应用的 views.py 文件中定义一个 user_login() 视图来处理表单数据。

  1. 如果是 HTTP POST 请求，提取表单信息：
    1. 使用 get() 方法提取 username、password 值，当对应值不存在时，返回 None；
    2. 使用 authenticate() 函数检查用户名和密码是否匹配数据库中已存在的某个用户账户。如果匹配成功，就返回一个 User 对象，否则返回 None；
    3. 如果匹配成功，查看用户账户是否是激活状态（由 User 模型的 is_active 决定）；如果账户为激活状态，使用 Django 提供 login() 函数登录用户，然后使用 HttpResponseRedirect 类重定向回首页，其中 reverse() 含函数获取应用首页的 URL；如果账户不是激活状态，使用 HttpResponse 类传递错误信息给用户。
    4. 如何匹配不成功，在终端打印错误信息并传递错误信息给用户。
  2. 如果不是 HTTP POST 请求，重新渲染登录模版。

登陆页面的 HTML 模版

最后要添加 URL 映射，并且在 HTML 模版中添加登陆链接。

注销功能

使用 Django 提供的 logout() 函数实现用户注销功能。

在应用的 views.py 文件中定义一个 user_logout() 视图来处理注销功能。

注销功能只有已登陆的用户才能使用，使用 login_required() 装饰器函数加以限制。

最后要添加 URL 映射，并且在 HTML 模版中添加注销链接。

限制访问不同内容

登陆/注册和注销链接根据当前是否有用户已登陆会有不同的显示，可以在 HTML 模版中 使用 {% if user.authenticated %} 模版标签来判断是否存在已登陆用户。
