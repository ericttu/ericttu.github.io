Django 内置模型字段类型

Posted on 2018-02-20 | In Django |

Django 模型是使用 Python 代码对数据库中数据的描述，是数据的结构，等效于 SQL 中的 CREATE TABLE 语句。

定义模型时，使用一个 Python 类（django.db.models.Model 的子类）表示一个模型，模型中的一个属性对应数据库中表的列，属性名对应列的名称，字段类型（如 CharField ）对应数据库列的类型（如 varchar）。

模型字段类

上面代码中定义了两个模型，在这两个模型中，使用了许多字段类型（如 CharField、 DateTimeField 等)，Django 内置了许多字段类型，常见的字段类和说明如下：

字段类 说明
AutoField 一个 IntegerField，根据 ID 自动递增。
IntegerField 整数。在Django 支持的所有数据库中取值范围是 -2147483648 到 2147483647；当 localize（支持表单数据输入以及渲染输出的本地化的参数）设置为 False 时，默认的表单部件是 NumberInput，否则是 TextInput。
CharField 用来存储各种长度的字符串， 默认表单部件 TextInput。
BooleanField 真假值字段，默认为默认表单部件 CheckboxInput。
DateField 日期，在 Python 中使用 datetime.date 实例表示。
DateTimeField 日期和时间，在 Python 中使用 datetime.datetime 实例表示。
EmailField 一种 CharField，使用EmailValidator 验证邮箱格式，max_length 的默认值为 254。
FileField 文件上传字段。
FilePathField 一种 CharField，限定只能在文件系统中的特定目录里选择文件。
FloatField 浮点数，在 Python 中使用 float 实例表示。当 localize 设置为 False 时，默认的表单部件是 NumberInput，否则是 TextInput。
ImageField 属性和方法继承自 FileField，对上传的对象进行了验证，增加了 height 和width 两个属性。需要 Pillow 库支持。
GenericIPAddressField IPV4或IPV6地址，字符串格式（如 192.168.1.1或fe80::e41c:c9da:a324:cc50%12），默认表单部件 TextInput。
SlugField 别名，Slug 字段只能包含字母，数字，下划线或者是连字符，通常用来作为短标签，放在URL中。可以使用这个字段用来保存节点的 URL，如果没有指定默认长度 max_length，长度默认为50。
TextField 大段文本字段，默认表单组件是 Texterea。
URLField 用于输入 URL 的 CharField。

常用字段属性

字段属性用于字段类型中对字段类型加以限制，如 CharField(max_length=127) 限制了字段类型的最大长度为 127。

常用通用字段属性

此类字段属性参数是全部字段类型都可以用的，而且是可以选择的。

  1. null：默认为 False，如果为True，Django 将在数据库中把空值存储为 null。

    基于字符串的字段，如 CharField 和 TextField，不应该使用null，因为空字符串值始终存储为空字符串，而不是 null。

  2. blank：默认为 False，如果为True，则表单验证时允许输入空值，否则为必填项。

  3. default：字段的默认值，可以是一个值或可调用对象，如果是一个可调用对象，每次这个模型类被创建时，它都会被调用。这个默认值不可以是一个可变对象（如：字典，列表等）。

  4. editable：默认为 True，如果为 False，字段不在管理后台或其他 ModelForm 中显示，验证模型时
    也会跳过。

  5. error_messages：用于覆盖字段抛出异常时的默认消息。值为一个字典，通过键指定想覆盖的错误消息。错误消息键包括 null、blank、invalid、invalid_choice、unique 和 unique_for_date。

  6. help_text：在表单小组件旁显示的额外帮助文本。即便不在表单中显示，也能用作文档。

  7. primary_key：设为 True 时，指定字段为模型的主键。如果模型中没有一个字段设定 primary_key=True，Django 会自动添加一个 AutoField 用于存储主键，主键字段是只读的。

  8. unique：设为 True 时，在表中字段的值必须是唯一的。除了 ManyToManyField、OneToOneField 和 FileField 之外其他字段都可以使用这个设置。

  9. verbose_name：一个字段的可读性更高的名称，如果未设定，Django 将使用字段的属性名称（下划线转换成空格）自动生成一个。

  10. validators：用于验证字段的验证器列表。

常用非通用字段属性

  1. max_length：字段的最大长度（字符个数），常用于 CharField、EmailField、TextField、SlugField、URLField。
  2. auto_now：每次保存对象时自动设为当前日期，常用于 DateField、DateTimeField。
  3. auto_now_add：创建对象时自动设为当前日期，常用于 DateField、DateTimeField。

模型字段关系

关于模型之间的关系，Django 提供了三个字段类型：ForeignKey（多对一关系）、OneToOneField（一对一关系）、ManyToManyField（多对多关系）。都需要一个位置参数：与该模型关联的类。

  1. 一个对象与自身有多对一关系，使用 models.ForeignKey(‘self’)。
  2. 与尚未定义的模型建立关系，不能使用模型对象，应该使用模型的名称：models.ForeignKey('Undefined')
  1. Django 会在该字段名称添加 “_id” 后缀，作为数据库列的名称。Node 模型中 category 的字段名称实际为 category_id。

  2. 三个字段类型可以接收一些参数，常用两个参数如下：

    • related_name：用于获取所关联对象的名称。当一个模型中有两个及以上属性同时引用另一个模型时，需要使用此参数，否则会出现重名问题。如果不想让 Django 创建逆向关系，把 related_name 设为 ‘+’，或者以 ‘+’ 结尾。

    • on_delete：外键引用的对象被删除时，设置包含外键的对象是否也被删除，on_delete 可取的值在 django.db.models 中定义:

      1. CASCADE：层叠删除，默认值。
      2. PROTECT：抛出 ProtectedError（django.db.IntegrityError 的子类），禁止删除被引用的对象。
      3. SET_NULL：把外键设为 null，仅当 null 为 True 时才能这么做。

      4. SET_DEFAULT：把外键设为默认值，必须为外键设定默认值。

        果关联的表中没有关联的记录，访问反向关系时会抛出 `DoesNotExist` 异常。

模型元选项

在上面的模型代码中，定义一个内部 Meta 类，用来对当前的模型类或模型字段进行设置约束。

常用模型元数据选项如下：

模型元数据 说明
abstract 如果为 True，表示模型类是一个抽象基类。
app_label 如果定义模型的应用不在 INSTALLED_APPS 中，必须指定所属的应用。
db_table 模型所用的数据表的名称，默认情况下，Django 使用表名后加一个 s 作为数据表的名称。
ordering 对象的默认排序，获取对象列表时使用，前面带有可选的 ‘ - ‘ 表示倒叙，没有表示正序，使用 ‘ ? ‘ 表示随机排序。
permissions 创建对象时写入权限表的额外权限。
default_permissions 默认为（’add’, ‘change’, ‘delete’）。
unique_together 设定组合在一起时必须唯一的多个字段名称。 unique_together = ('字段1', '字段2') 是一个字段的元组，组合起来的时候代表里面的字段分别都是唯一的。
verbose_name 设定对象的单数名称。
verbose_name_plural 设定对象的复数名称。
